Front-end checklist Quick overview of things you should not forget on your website front-end side

Global

  • Favicons (1)

    A favicon.ico file is at the root of the web folder

  • Favicons (2)

    A set of touch icon is define in the 

    <head>
    and present into the server

  • Fonts

    Webfonts licence is handle

  • Browsers tests desktop & mobile

    A set a target browsers & devices is define

  • Browsers tests desktop & mobile (2)

    Tests are done!

  • Share

    Facebook & twitter metas tags are added on each page

  • No broken links

    Broken links are tested

    http://validator.w3.org/checklink http://iwebtool.com/broken_link_checker

  • 404

    a 404 page exist

HTML

  • doctype

    Each page begins with a doctype.

  • html validation

    All pages validate throw validator.w3.org (Errors and warnings have to be assumed)

  • title

    Each page has a unique comprehensive title

  • html semantic

    markup is correctly write in regards of titles, lists, no table for layout

  • unique ID

    In html code, each id attributes are unique

  • Tables (th, caption...)

    HTML tables are build in a semantic and accessible way

  • Forms (label, fieldset...)

    HTML forms are build in a semantic and accessible way

CSS

  • reset

    Project should begins with a reset or normalize

  • minification

    On production, css files are minified.

  • files structure (sass)

  • css validator

    All pages validate throw jigsaw.w3.org (Errors and warnings have to be assumed)

  • :hover, :focus

    Mostly, hover declaration should duplicate with focus

  • avoid !important

  • avoid @import

JS

  • concatenation

    On production, as possible, javascript files are concatenate

  • minification

    On production, as possible, javascript files are minified

  • vendor libraries

    a

Images

  • retina ready?

    Are assets ready for retina devices (svg, double size png)

  • SEO ready names

    Images with SEO meanings are corectly named (-,_)

  • No text

    Images should not include texts

  • Weight

    An optimization tool has been used

Performance

  • pagespeed test

    At least one test of performance is done

  • css head

    Css files are declared on

  • js at the bottom

    As possible, javascript are located at the bottom at each document.

    => avoid to block page rendering

  • no inline script & style

  • img width/height

    If images are not resized, indicate width and height attributes

  • Nb elements DOM

    Reduce html markup

    => Page load & browsers code cross

  • start render

  • Gzip

    Is gzip activated on production ?

Accessibility

  • img alt

    Alternative text images

Code

  • indentation

  • double quotes
